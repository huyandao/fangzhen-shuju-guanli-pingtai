# 仿真数据管理工具维护指南

本文档面向后续一起维护本项目的工程师。当前实现是一个轻量级原型，前端集中在 `frontend/`，后端集中在 `backend/`，根目录配置文件负责运行路径和端口。

技术边界：

- 前端使用 HTML / CSS / JavaScript，不引入前端构建工具。
- 后端使用 Python 标准库，保持工作站免安装依赖即可运行。

## 代码入口

- `frontend/index.html`：主应用，包含界面、状态管理、图谱逻辑、附件逻辑。
- `backend/server.py`：工作站局域网服务端，负责页面托管、状态保存、附件上传下载删除。
- `config.json`：运行配置，集中设置 `host`、`port`、`frontend_index`、`data_dir`。
- `run.py`：项目根目录启动入口，默认读取 `config.json`。
- `README.md`：运行方式和局域网部署说明。

## 前端结构

`frontend/index.html` 的 `<script>` 已按维护区块组织：

- 常量与示例数据：默认模板、零件、卡片、图谱边。
- 状态规范化与持久化：兼容旧数据，本地模式用 `localStorage`，工作站模式用 `/api/state`。
- 附件管理：本地模式使用 File System Access API，工作站模式上传到 `server_data/attachments`。
- 图谱数据模型与查询：卡片查找、显式边、由 `parentId` 推导的隐式边、环路检测。
- 渲染入口：`renderAll()` 是全量刷新入口。
- 右侧面板：详情、编辑两个页面；模板编辑页在中间工作区显示。
- 模板面板渲染与保存：模板字段迁移每张卡片的 `values`。
- 业务动作：新建、复制、自动布局、导入导出。
- DOM 事件绑定：顶部按钮、拖拽、缩放、文件导入。

## 核心状态模型

主状态对象是 `state`，主要字段：

- `parts`：左侧仿真数据库里的零件列表。
- `template.fields`：卡片模板字段。`code` 是稳定内部键，`name` 是界面显示名。
- `template.fields[].type`：支持 `number`、`text`、`image`。图片字段保存文件元数据，文件本体仍走附件归档目录。
- `card.files[]`：普通关联文件元数据。PDF 文件名可打开新标签预览，所有真实归档文件都提供下载入口。
- `cards`：仿真卡片。模板字段值保存在 `card.values[field.code]`。
- `edges`：显式图谱关系边，保存关系类型。
- `card.parentId`：卡片的主要来源节点。没有显式边时会由它生成隐式边。
- `selectedIds`：底部对比表选中的卡片。

用户账号保存在 `server_data/users.json`，明文保存账号、密码、角色和启用状态。首次运行自动创建 `admin / admin123` 管理员账号。

维护原则：

- 修改模板字段时优先保留 `code`，否则历史数据需要迁移。
- 修改来源关系时必须调用 `syncParentEdge()`，避免 `parentId` 和 `edges` 不一致。
- 新增保存逻辑时不要绕过 `renderAll()` 或 `saveState()`，否则本地和工作站模式可能表现不同。

## 服务端约定

`backend/server.py` 提供以下接口：

- `POST /api/login`
- `POST /api/logout`
- `GET /api/me`
- `GET /api/users`：管理员查看用户列表。
- `POST /api/users`：管理员新增用户。
- `PUT /api/users/<id>`：管理员修改显示名、密码、角色、启用状态。
- `DELETE /api/users/<id>`：管理员删除用户。
- `GET /api/state`
- `PUT /api/state`
- `POST /api/upload?cardId=...&folderName=...`
- `GET /api/attachments/<relative-folder>/<file>`：普通访问用于 PDF 预览；加 `?download=1` 时强制下载。
- `DELETE /api/attachments/<relative-folder>/<file>`
- `DELETE /api/attachments/<relative-folder>`

数据目录：

- `server_data/state.json`：平台状态。
- `server_data/users.json`：用户账号、明文密码、角色和启用状态。
- `server_data/attachments/`：附件归档目录。可通过修改 `config.json` 的 `data_dir` 把整套运行数据改到指定磁盘路径。
- `server_data/attachments/零件类型/零件编号_零件名称/卡片编号_卡片名称_卡片ID/`：卡片级目录，存放 `card_detail.json`、图片字段文件和关联附件。

`server_data/` 不应提交到仓库。

## 常见修改入口

- 改默认示例卡片：修改 `defaultState.cards`。
- 改模板默认字段：修改 `defaultState.template.fields` 和 `normalizeTemplate()`。
- 改详情页展示：修改 `renderDetailPanel()`。
- 改编辑页表单：修改 `renderEditPanel()`。
- 改模板页：修改 `renderTemplatePanel()` 和 `applyTemplateDraft()`。
- 改图谱布局：修改 `autoLayout()`、`updateCanvasBounds()`、`renderLinks()`、`renderNodes()`。
- 改附件行为：修改 `uploadAttachments()`、`deleteAttachment()`、`deleteCardAttachmentFolder()`。
- 改运行路径和端口：修改根目录 `config.json`。
- 改服务端存储：修改 `backend/server.py` 中的 `handle_get_state()`、`handle_put_state()`、`handle_upload()`。

## 验证命令

每次修改后建议至少运行：

```bash
awk '/<script>/{flag=1;next}/<\/script>/{flag=0}flag' 'frontend/index.html' | node --check
python3 -m py_compile backend/server.py run.py
python3 run.py --help
git diff --check -- frontend/index.html backend/server.py run.py config.json README.md docs/维护指南.md
```

如果本机没有默认 `node`，可使用 Codex 环境里的 Node 路径执行同样的 `--check`。

## 协作注意事项

- 不要提交 `server_data/`，这里是工作站运行数据和归档附件。
- 不要把临时交互状态持久化为业务状态，例如 `createChildMode`、`pendingConnectId`。
- 服务端保存是串行防抖的，修改保存逻辑时要保持“最后一次状态最后落盘”。
- 附件删除要同时更新卡片元数据和磁盘文件，避免出现孤立附件。
