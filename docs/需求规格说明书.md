# 仿真数据管理平台需求规格说明书

版本：V0.4
日期：2026-07-06
适用阶段：试用版单机/局域网部署、MVP 功能验收

## 1. 项目概述

### 1.1 背景

折叠方向盘等结构件开发过程中，结构方案会频繁迭代。每次结构变更后，仿真工程师需要记录仿真输入、关键结果、结论、报告和相关附件，并说明该结果与前后版本之间的关系。

本平台用于沉淀仿真数据和结果演化关系，重点解决以下问题：

- 仿真结果卡片字段不统一，后续难以对比。
- 仿真附件散落在本地文件夹，难以追溯。
- 结构迭代、材料替换、参数调整等关系难以直观看清。
- 多个仿真零件和多张结果卡片缺少统一管理入口。

平台不负责仿真求解，不替代 Abaqus、HyperMesh、ANSA 等仿真工具；当前试用版定位为工作站上运行的轻量数据管理工具。

### 1.2 当前建设目标

- 在一台工作站上运行 Python 后端，局域网用户通过浏览器访问。
- 使用前端 HTML / CSS / JavaScript 和 Python 标准库后端，尽量减少部署依赖。
- 支持用户登录、角色识别和管理员用户管理。
- 支持管理不同仿真零件库，并在零件库下维护仿真结果卡片。
- 支持自定义卡片模板字段，包括数字、文本、图片字段。
- 支持创建、编辑、查看仿真结果卡片。
- 支持通过指定父节点创建子卡片，自动形成结果图谱关系。
- 支持编辑卡片来源和连线关系类型，并同步更新图谱连接。
- 支持附件上传、PDF 预览、附件下载和随卡片删除清理本地归档文件。
- 支持图片字段上传、详情页缩略图展示和点击放大预览。
- 支持图谱自动布局、画布范围自适应、当前卡片关系聚焦展示。
- 支持数据、用户、附件统一保存到服务端配置的数据目录。

### 1.3 当前非目标

- 不做仿真求解。
- 不解析 odb、inp、data 等大型仿真结果文件内容。
- 不做在线三维模型预览。
- 不做企业级 SSO、复杂权限矩阵、多组织隔离。
- 不做完整 OA 审批流、消息通知和任务工单闭环。
- 不做 AI 对话、自动优化建议和附件内容智能解析。
- 试用版不做密码哈希，账号密码以明文保存在工作站数据目录中，仅适合内网试用。

## 2. 技术和部署范围

### 2.1 技术栈

- 前端：`frontend/index.html`，使用 HTML / CSS / JavaScript。
- 后端：`backend/server.py`，使用 Python 标准库 `http.server` 实现接口和静态文件托管。
- 启动入口：项目根目录 `run.py`。
- 运行配置：项目根目录 `config.json`。
- 数据目录：默认 `server_data/`，可通过 `config.json` 或启动参数调整。

### 2.2 部署方式

平台部署在一台工作站上，启动命令：

```bash
python3 run.py
```

默认监听：

```json
{
  "host": "0.0.0.0",
  "port": 8088,
  "frontend_index": "frontend/index.html",
  "data_dir": "server_data"
}
```

访问方式：

- 本机访问：`http://127.0.0.1:8088/`
- 局域网访问：`http://工作站局域网IP:8088/`

### 2.3 保存方式

服务端模式下，数据保存在工作站统一目录：

- 平台状态：`server_data/state.json`
- 用户账号：`server_data/users.json`
- 附件归档：`server_data/attachments/`

附件按以下规则分层保存：

```text
server_data/attachments/
└── 零件类型/
    └── 零件编号_零件名称/
        └── 卡片编号_卡片名称_卡片ID/
            ├── card_detail.json
            ├── 图片字段文件
            └── 关联附件文件
```

直接双击 `frontend/index.html` 时仍可进入本地模式，本地模式用于原型查看，不作为多人局域网部署方式。

## 3. 用户角色

### 3.1 管理员

管理员负责平台配置和基础数据维护。

当前已实现能力：

- 登录平台。
- 创建、编辑、删除用户。
- 设置用户角色和启用状态。
- 编辑卡片模板。
- 创建、编辑、删除仿真卡片。
- 编辑图谱关系和卡片来源。
- 上传、删除附件和图片字段。
- 恢复示例数据。

限制：

- 系统必须至少保留一个启用状态的管理员。
- 账号密码明文保存在 `server_data/users.json`。

### 3.2 仿真工程师

仿真工程师负责维护仿真结果卡片、图谱和附件。

当前已实现能力：

- 登录平台。
- 查看结果图谱和卡片详情。
- 创建、编辑仿真卡片。
- 指定父节点创建子卡片。
- 编辑卡片来源和连接关系类型。
- 上传、下载、删除卡片附件。
- 查看 PDF 附件预览。
- 参与多卡片对比。

当前限制：

- 不能编辑模板。
- 不能管理用户。

### 3.3 结构工程师

结构工程师主要查看仿真结果、图谱和附件元数据。

当前已实现能力：

- 登录平台。
- 查看仿真数据库、结果图谱、卡片详情和多卡片对比。
- 查看卡片关联文件名称、后缀和元数据。

当前限制：

- 不能编辑模板。
- 不能管理用户。
- 不能编辑图谱、卡片内容和附件。

## 4. 核心业务对象

### 4.1 仿真零件库

仿真零件库是左侧数据库菜单中的管理对象，用于按零件组织仿真卡片。

字段建议：

| 字段 | 说明 |
|---|---|
| `id` | 零件唯一标识 |
| `code` | 零件编号 |
| `name` | 零件名称 |
| `type` | 零件类型 |
| `owner` | 负责人 |

当前要求：

- 左侧以数据库/文件树样式展示不同仿真零件。
- 支持新建仿真零件。
- 图谱区域只展示当前选中零件相关卡片；选择全部时展示全部卡片。
- 左侧仿真数据库标签不显示图谱关系文字。

### 4.2 卡片模板

卡片模板定义仿真结果卡片有哪些字段，以及字段如何展示。

模板字段当前支持：

| 字段配置 | 当前状态 |
|---|---|
| 字段名称 | 已支持 |
| 字段编码 | 已支持，用作系统内部稳定键和数据迁移键 |
| 类型：数字 | 已支持 |
| 类型：文本 | 已支持 |
| 类型：图片 | 已支持 |
| 单位 | 已支持 |
| 必填 | 已支持 |
| 节点摘要 | 已支持 |
| 详情摘要 | 已支持 |
| 对比字段 | 已支持，图片字段不参与对比 |
| 关键数值 | 已支持，主要用于数字字段 |
| 结论内容 | 已支持 |

产品规则：

- 只有管理员可以进入模板页面和保存模板。
- 模板按钮位于页面顶部搜索框左侧。
- 进入模板页面时，右侧卡片详情栏隐藏，中间主界面显示模板编辑页。
- 模板修改后需要点击保存按钮才应用。
- 保存模板后，已有卡片的字段值会随字段编码迁移和补齐。
- 字段编码用于稳定保存历史卡片值，用户不需要频繁修改。

### 4.3 仿真结果卡片

仿真结果卡片是平台的核心记录对象。

建议字段：

| 字段 | 说明 |
|---|---|
| `id` | 卡片唯一标识 |
| `partId` | 所属仿真零件 |
| `parentId` | 来源卡片，即图谱父节点 |
| `relationType` | 与来源卡片的关系类型 |
| `code` | 卡片编号 |
| `title` | 卡片名称 |
| `status` | 卡片状态 |
| `owner` | 负责人 |
| `task` | 仿真任务或说明 |
| `model` | 模型信息 |
| `report` | 报告信息 |
| `values` | 模板字段值 |
| `change` | 结构或参数变更说明 |
| `conclusion` | 仿真结论 |
| `files` | 关联附件列表 |
| `x` / `y` | 图谱画布位置 |
| `placed` | 是否已放入图谱 |

当前卡片状态建议：

```text
recommended  推荐
passed       已完成
running      仿真中
failed       未达标
archived     已归档
```

### 4.4 卡片附件

附件用于保存仿真输入、结果、报告、表格、图片等文件。

当前要求：

- 支持上传本地不同格式文件。
- 上传后文件复制到服务端配置的数据目录。
- 文件按零件库和卡片目录归档。
- 卡片删除时，对应本地归档附件目录也随之删除。
- PDF 文件名称可点击，并在新浏览器页面中预览。
- 其他格式文件只显示文件名称和后缀。
- 每个文件后显示下载按钮。
- 关联文件元数据保存在卡片 `files` 中。
- 卡片目录中保存 `card_detail.json` 快照，便于人工检查归档内容。

附件元数据建议：

| 字段 | 说明 |
|---|---|
| `id` | 附件唯一标识 |
| `name` | 原始文件名 |
| `storedName` | 归档文件名 |
| `folderName` | 相对归档目录 |
| `relativePath` | 相对保存路径 |
| `type` | MIME 类型 |
| `size` | 文件大小 |
| `uploadedAt` | 上传时间 |
| `serverStored` | 是否已保存在服务端 |
| `url` | 服务端访问地址 |

### 4.5 图片字段

模板字段类型为图片时，卡片详情和编辑页应支持图片上传和查看。

当前要求：

- 支持格式：`jpg`、`jpeg`、`gif`、`png`、`bmp`。
- 图片文件同样保存到当前卡片归档目录。
- 详情页显示缩略图。
- 点击缩略图后弹出放大预览。
- 图片字段不参与多卡片对比表。

### 4.6 结果图谱

结果图谱用于表达仿真卡片之间的来源和演化关系。

当前产品逻辑：

- 点击“新建卡片”按钮后，进入新建子卡片模式。
- 鼠标状态变化，提示用户选择图谱中的父卡片。
- 点击某个图谱卡片后，在该卡片子集中创建新卡片。
- 新卡片自动设置 `parentId`，并生成对应连接关系。
- 卡片详情/编辑页修改来源卡片后，图谱连接关系同步变化。
- 编辑页面可以定义连线关系类型，例如结构迭代、材料替换、参数调整。
- 图谱只聚焦显示当前选中卡片的连接关系，减少画面混乱。
- 自动布局间距应参考原型图，不宜过度紧凑。
- 画布空间范围上限根据卡片位置自动调整，避免上下空间不足导致卡片重叠。

关系类型当前建议：

```text
来源于
结构迭代
材料替换
参数调整
减重优化
横向对比
复核结果
推荐路径
```

### 4.7 多卡片对比

多卡片对比用于横向比较不同仿真结果。

当前要求：

- 支持在图谱节点上勾选多个卡片。
- 对比字段来自模板中勾选“对比”的字段。
- 图片字段不进入对比表。
- 支持清空对比。
- 支持复制对比表文本。

## 5. 页面和交互需求

### 5.1 登录页

当前要求：

- 访问服务端模式时必须登录。
- 默认管理员账号：

```text
账号：admin
密码：admin123
角色：管理员
```

- 登录成功后进入主应用。
- 前端根据后端 `/api/me` 返回的用户角色控制按钮显示和编辑权限。

### 5.2 顶部栏

当前要求：

- 顶部显示平台名称、当前用户和角色。
- “模板”和“仿真结果图谱”按钮位于搜索框左侧。
- 管理员可见“用户管理”按钮。
- 结构工程师进入后，模板按钮不可用。

### 5.3 左侧仿真数据库

当前要求：

- 左侧以数据库菜单/文件树样式展示仿真零件。
- 支持全部卡片视图。
- 支持新建仿真零件。
- 展示每个零件下的卡片数量和附件数量。
- 不在左侧显示图谱关系。

### 5.4 中间主界面

当前要求：

- 默认显示仿真结果图谱。
- 图谱区域应占据主要界面比例。
- 进入模板页面时，中间显示模板编辑页面。
- 模板页面隐藏右侧卡片详情栏。
- 图谱节点可点击选择，修复后应允许从一个卡片切换选择到另一个卡片。
- 画布支持缩放、自动布局和重置视图。

### 5.5 右侧卡片详情

当前要求：

- 未选择卡片时显示空状态。
- 选择卡片后显示详情摘要、关键数值、结论内容、图片字段、变更说明和附件。
- 可切换到编辑状态修改卡片内容。
- 管理员和仿真工程师可以编辑。
- 结构工程师只读。

### 5.6 用户管理弹窗

当前要求：

- 仅管理员可见。
- 可新增用户。
- 可修改显示名、角色、启用状态和密码。
- 可删除用户。
- 不允许删除或禁用最后一个启用管理员。

## 7. 后端接口需求

当前后端接口：

| 方法 | 路径 | 说明 | 权限 |
|---|---|---|---|
| `POST` | `/api/login` | 登录 | 公开 |
| `POST` | `/api/logout` | 退出登录 | 登录用户 |
| `GET` | `/api/me` | 获取当前用户 | 登录用户 |
| `GET` | `/api/users` | 用户列表 | 管理员 |
| `POST` | `/api/users` | 新增用户 | 管理员 |
| `PUT` | `/api/users/<id>` | 修改用户 | 管理员 |
| `DELETE` | `/api/users/<id>` | 删除用户 | 管理员 |
| `GET` | `/api/state` | 获取平台状态 | 登录用户 |
| `PUT` | `/api/state` | 保存平台状态 | 管理员、仿真工程师 |
| `POST` | `/api/upload?cardId=...&folderName=...` | 上传附件 | 管理员、仿真工程师 |
| `GET` | `/api/attachments/<relative-folder>/<file>` | 预览或访问附件 | 登录用户 |
| `GET` | `/api/attachments/<relative-folder>/<file>?download=1` | 下载附件 | 登录用户 |
| `DELETE` | `/api/attachments/<relative-folder>/<file>` | 删除单个附件 | 管理员、仿真工程师 |
| `DELETE` | `/api/attachments/<relative-folder>` | 删除卡片附件目录 | 管理员、仿真工程师 |

会话要求：

- 登录成功后服务端写入 `simulation_session` Cookie。
- 会话保存在服务端内存中。
- 服务重启后用户需要重新登录。

## 8. 数据模型

### 8.1 服务端文件

```text
server_data/
├── state.json
├── users.json
└── attachments/
```

### 8.2 用户对象

```json
{
  "id": "u-admin",
  "username": "admin",
  "password": "admin123",
  "displayName": "系统管理员",
  "role": "admin",
  "enabled": true,
  "createdAt": "2026-07-06T00:00:00"
}
```

角色枚举：

```text
admin
simulation
structure
```

### 8.3 平台状态对象

`state.json` 主要包含：

```text
parts
template
cards
edges
activePartId
activeCardId
selectedIds
filter
zoom
```

### 8.4 模板字段对象

```json
{
  "name": "一阶模态",
  "code": "first",
  "type": "number",
  "unit": "Hz",
  "required": true,
  "showOnNode": true,
  "detailSummary": true,
  "compare": true,
  "keyMetric": true,
  "conclusionField": false
}
```

### 8.5 图谱边对象

```json
{
  "id": "edge-a-b",
  "from": "card-a",
  "to": "card-b",
  "type": "结构迭代"
}
```

规则：

- `card.parentId` 是卡片主要来源。
- `edges` 保存显式连接和关系类型。
- 修改来源卡片时，必须同步更新 `edges`。

## 11. 验收标准

### 11.1 启动和访问

- 在工作站执行 `python3 run.py` 后服务正常启动。
- 本机可以访问 `http://127.0.0.1:8088/`。
- 同一局域网设备可以访问 `http://工作站IP:8088/`。
- 首次运行自动创建默认管理员账号。

### 11.2 用户和权限

- 默认管理员可以登录。
- 管理员可以打开用户管理弹窗。
- 管理员可以新增、编辑、禁用和删除用户。
- 非管理员看不到或不能使用模板编辑和用户管理入口。
- 结构工程师只能查看，不能编辑卡片和图谱。

### 11.3 模板

- 管理员可以进入模板页面。
- 进入模板页面后右侧详情栏隐藏。
- 模板保存后卡片编辑字段同步变化。
- 图片字段可以上传并在详情页显示缩略图。

### 11.4 卡片和图谱

- 点击新建卡片后进入选择父节点模式。
- 点击父卡片后，新卡片成为其子卡片。
- 修改卡片来源后，图谱连接同步变化。
- 可以编辑关系类型。
- 选择一个卡片后，仍可继续选择其他卡片。
- 图谱布局不应过度拥挤，画布范围应能容纳卡片。

### 11.5 附件

- 上传附件后，文件出现在服务端 `server_data/attachments` 对应卡片目录。
- PDF 文件名可点击并在新页面预览。
- 所有关联文件都有下载按钮。
- 删除卡片后，对应附件目录被删除。
- 服务重启后，卡片和附件仍可继续查看。
